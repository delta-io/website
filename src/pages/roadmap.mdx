This is the proposed Delta Lake 2022 H1 roadmap discussion thread. Below are the initially proposed items for the roadmap to be completed by June 2022. We will also be sending out a survey (we will update this issue with the survey) to get more feedback from the Delta Lake community!

## Performance Optimizations

---

Based on the overwhelming feedback from the Delta Users [Slack](https://go.delta.io/delta-users), [Google Groups](https://groups.google.com/forum/#!forum/delta-users), Community AMAs (on [Delta Lake YouTube](http://youtube.com/c/deltalake)), Delta Lake 2021H2 survey, and [2021H2 roadmap](https://github.com/delta-io/delta/issues/748), we propose the following Delta Lake performance enhancements in the next two quarters.

| Issue                                                 | Description                                                                                                                                                                             | Target CY2022                                                |
| :---------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------- |
| [927](https://github.com/delta-io/delta/issues/927)   | **OPTIMIZE (file compaction)**: Table optimize is an operation to rearrange the data and/or metadata to speed up queries and/or reduce the metadata size                                | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [923](https://github.com/delta-io/delta/issues/923)   | **File skipping using columns stats**: This is a performance optimization that aims at speeding up queries that contain filters (WHERE clauses) on non-partitionBy columns.             | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [931](https://github.com/delta-io/delta/issues/931)   | **Automatic data skipping using generated columns**: Enhance [generated columns](https://docs.delta.io/1.0.0/delta-batch.html#use-generated-columns) to include automatic data skipping | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [1134](https://github.com/delta-io/delta/issues/1134) | **OPTIMIZE ZORDER**: Data clustering via multi-column locality-preserving space-filling curves with offline sorting.                                                                    | Q2                                                           |
|                                                       | **MERGE Performance Improvements**: We will be providing a project improvement plan (PIP) document shortly on the proposed design for discussion.                                       | Q2/Q3                                                        |

## Schema Operations

---

For this year, our focus will be on columnar mappings.

| Issue                                               | Description                                                                                                                                                                              | Target CY2022                                                |
| :-------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------- |
| [958](https://github.com/delta-io/delta/issues/958) | **Support for renaming column**: Rename column with ALTER TABLE                                                                                                                          | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [957](https://github.com/delta-io/delta/issues/957) | **Support for arbitrary column names**: Support characters in column names not allowed by Parquet                                                                                        | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
|                                                     | **Support for dropping columns**: Drop column with ALTER TABLE                                                                                                                           | Q2                                                           |
| [348](https://github.com/delta-io/delta/issues/348) | **Support for dynamic partition overwrite**: Currently you can overwrite using the `replaceWhere` option but in various scenarios, it is more convenient to specify overwrite partition. | Q2                                                           |

## Operations Enhancements

---

Two very popular requests are planned for this semester: Table Restore, S3 multi-cluster writes.

| Issue                                                                                                | Description                                                                                                                                                                                                                                                                                                              | Target CY2022                                                |
| :--------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------- |
| [903](https://github.com/delta-io/delta/pull/903), [863](https://github.com/delta-io/delta/pull/863) | **Table Restore**: Rollback to a previous version of a Delta table using Python, Scala, and/or SQL APIs.                                                                                                                                                                                                                 | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [41](https://github.com/delta-io/delta/issues/41)                                                    | **S3 multi-cluster writes**: Allows multiple clusters/drivers/JVMs to concurrently write to S3 using DynanoDB as the lock store. Please refer to this PIP: [[2021-12-22] Delta OSS S3 Multi-Cluster Writes](https://docs.google.com/document/d/1Gs4ZsTH19lMxth4BSdwlWjUNR-XhKHicDvBjd2RqNd8/edit#heading=h.mjjuxw9mcz9h) | [1.2](https://github.com/delta-io/delta/releases/tag/v1.2.0) |
| [747](https://github.com/delta-io/delta/issues/747)                                                  | **delta.io.Guide**: Enhance the Delta Lake documentation by creating a new guide (PIP will follow soon)                                                                                                                                                                                                                  | Q2/Q3                                                        |
|                                                                                                      | **Iceberg to Delta Converter**: Ability to convert Iceberg table to Delta table without a rewrite.                                                                                                                                                                                                                       | Q3                                                           |
|                                                                                                      | **Table Cloning**: Clones a source Delta table to a target destination at a specific version. A clone can be either deep or shallow: deep clones copy over the data from the source and shallow clones do not.                                                                                                           | Q3                                                           |
| [1105](https://github.com/delta-io/delta/issues/1105)                                                | **Change Data Feed**: The Delta change data feed represents row-level changes between versions of a Delta table. When enabled on a Delta table, the runtime records “change events” for all the data written into the table.                                                                                             | Q3                                                           |
